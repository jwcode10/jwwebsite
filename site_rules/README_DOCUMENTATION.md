# Journey Well - Documentation Index

## Overview
This directory contains all essential documentation for building and maintaining the Journey Well website. **All agents must read the relevant documentation before starting any work.**

## Documentation Files

### 1. `SITE_STRUCTURE.md`
**Purpose**: Defines site architecture, URL hierarchy, page relationships, and navigation structure.

**When to Use**:
- Before creating any new page
- When setting up routing
- When planning internal linking
- When updating navigation

**Key Contents**:
- Complete site hierarchy
- URL slug conventions
- Internal linking strategy
- Page status tracking

---

### 2. `KEYWORDS_SEO.md`

**When to Use**:
- Before writing any page content
- When optimizing for SEO
- When creating meta titles/descriptions
- When planning content structure

**Key Contents**:
- Main and supporting keywords per page
- Search intent definitions
- SEO best practices checklist
- Keyword density guidelines

- Build priority order

## Workflow for Building Pages

### Step 1: Planning
1. ✅ Read `PHASED_ROLLOUT.md` to confirm page priority
2. ✅ Read `SITE_STRUCTURE.md` to understand URL and structure
3. ✅ Read `KEYWORDS_SEO.md` for keyword requirements
4. ✅ Read `DESIGN_PRD.md` for styling guidelines

### Step 2: Content
1. ✅ Use `HOMEPAGE_CONTENT.md` as tone/style reference
2. ✅ Integrate keywords from `KEYWORDS_SEO.md` naturally
3. ✅ Follow content structure from relevant documentation

### Step 3: Implementation
1. ✅ Create page with correct URL slug (`SITE_STRUCTURE.md`)
2. ✅ Apply design system (`DESIGN_PRD.md`)
3. ✅ Add required internal links (`SITE_STRUCTURE.md`)
4. ✅ Implement SEO requirements (`KEYWORDS_SEO.md`)

### Step 4: Quality Assurance
1. ✅ Verify all checklist items from `PHASED_ROLLOUT.md`
2. ✅ Test responsive design (`DESIGN_PRD.md`)
3. ✅ Validate SEO requirements (`KEYWORDS_SEO.md`)
4. ✅ Check internal linking (`SITE_STRUCTURE.md`)
